Connection Methods:

  • Direct: Services communicate via internal Kubernetes cluster DNS (e.g., http://backend:3000). Used when all services run in K8s. Webhooks work because services can reach each other via cluster DNS.
  • Port Forward: Accessing K8s services via localhost port forwarding. Services still run in K8s, but accessed locally. Webhooks may not work unless backend has LoadBalancer or ngrok URL.
  • Ngrok: Public HTTPS URLs via ngrok tunnels. All services run in K8s, but exposed via ngrok public URLs. Webhooks work because backend has a public HTTPS URL.
  • Telepresence: Local services intercepted into the cluster. Services run locally but appear as if they're in K8s. Used for local debugging with full cluster integration. Webhooks work because Telepresence makes local services accessible to the cluster.
  • Local: Service running on your machine (not in K8s). Code changes are instant with hot reload.
  • Waiting for Status: Backend hasn't reported connection method yet (Socket.IO not connected).
  • Socket Not Connected: WebSocket connection not established.

Note: Updates work via Socket.IO even when services are on K8s. Connection method changes are shown with notifications.
